Introduction to Functions

  • Functions are reusable blocks of code designed to perform a specific task.
  • They help in making code modular, reusable, and easier to manage.
  • Functions can take inputs (parameters) and return outputs (return values).

Function Declaration

  • Defines a function with a name, a list of parameters, and a block of code.
  • Can be called before its declaration due to hoisting.

function greet(name) {
    return `Hello, ${name}!`;
}

console.log(greet("Alice")); // Output: Hello, Alice!

Function Expression

  • Defines a function and assigns it to a variable.
  • Not hoisted; can only be used after the assignment.

const add = function(a, b) {
    return a + b;
};

console.log(add(2, 3)); // Output: 5

Arrow Functions

  • Concise syntax for writing functions, introduced in ES6.
  • Does not have its own this context, which makes them great for callbacks.

const multiply = (a, b) => a * b;

console.log(multiply(4, 5)); // Output: 20

Parameters and Arguments

  • Parameters are placeholders listed in the function definition.
  • Arguments are the actual values passed to the function when called.

function subtract(a, b) {
    return a - b;
}

console.log(subtract(10, 4)); // Output: 6

Return Values

  • A function can return a value using the return statement.
  • Once a return statement is executed, the function stops running and returns the specified value.

function square(x) {
    return x * x;
}

console.log(square(6)); // Output: 36

Default Parameters

  • Allows parameters to have default values if no argument is passed.
  • Introduced in ES6.

function greet(name = "Guest") {
    return `Hello, ${name}!`;
}

console.log(greet()); // Output: Hello, Guest!
console.log(greet("Bob")); // Output: Hello, Bob!

Function Scope

  • Variables declared inside a function are only accessible within that function.
  • Functions can access variables from their outer scope (lexical scoping).

function outer() {
    let outerVar = "I'm outside!";
    
    function inner() {
        let innerVar = "I'm inside!";
        console.log(outerVar); // Can access outer scope
    }

    inner();
    // console.log(innerVar); // Error: innerVar is not defined
}

outer();

IIFE (Immediately Invoked Function Expression)

  • A function that runs as soon as it is defined.
  • Often used to create a private scope and avoid polluting the global scope.

(function() {
    console.log("This is an IIFE!");
})(); // Output: This is an IIFE!

Summary

  • Functions are reusable blocks of code that can take inputs and return outputs.
  • They can be defined as declarations, expressions, or arrow functions.
  • Functions help keep your code organized, modular, and manageable.
