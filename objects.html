Introduction to Objects

  • Objects are collections of key-value pairs where keys are strings (or Symbols) and values can be any data type.
  • Objects are fundamental in JavaScript, providing a way to group related data and functionality.
  • They are used to model real-world entities, data structures, and more.

Creating Objects

  • Objects can be created using object literals, the new Object() syntax, or constructor functions.

// Using object literal
let person = {
    name: "Alice",
    age: 30,
    greet: function() {
        console.log("Hello!");
    }
};

// Using new Object()
let car = new Object();
car.brand = "Toyota";
car.model = "Camry";

// Using constructor function
function Animal(type) {
    this.type = type;
}
let cat = new Animal("Mammal");

Accessing Object Properties

  • Properties can be accessed using dot notation or bracket notation.
  • Bracket notation allows for dynamic property access and property names with special characters.

let book = {
    title: "1984",
    author: "George Orwell",
    "published-year": 1949
};

// Dot notation
console.log(book.title); // Output: 1984

// Bracket notation
console.log(book["published-year"]); // Output: 1949

// Dynamic property access
let key = "author";
console.log(book[key]); // Output: George Orwell

Modifying Object Properties

  • Object properties can be added, updated, or deleted dynamically.
  • Use dot or bracket notation to modify properties.

let user = {
    username: "john_doe",
    email: "john@example.com"
};

// Adding a property
user.age = 28;

// Updating a property
user.email = "john.doe@example.com";

// Deleting a property
delete user.username;

console.log(user); // Output: { email: "john.doe@example.com", age: 28 }

Object Methods

  • Methods are functions stored as object properties.
  • They allow objects to have behavior in addition to data.
  • Use the this keyword to refer to the current object within methods.

let calculator = {
    a: 5,
    b: 3,
    add: function() {
        return this.a + this.b;
    },
    subtract: function() {
        return this.a - this.b;
    }
};

console.log(calculator.add()); // Output: 8
console.log(calculator.subtract()); // Output: 2

Object Destructuring

  • Destructuring allows you to unpack properties from objects into distinct variables.
  • Simplifies code and makes it easier to work with objects.

let person = {
    name: "Bob",
    age: 25,
    country: "USA"
};

let { name, age } = person;

console.log(name); // Output: Bob
console.log(age); // Output: 25

Object Iteration

  • Use for...in loop to iterate over the keys of an object.
  • Use Object.keys(), Object.values(), and Object.entries() for different types of iteration.

let settings = {
    theme: "dark",
    notifications: true,
    autoSave: false
};

// Using for...in
for (let key in settings) {
    console.log(`${key}: ${settings[key]}`);
}

// Using Object.keys()
Object.keys(settings).forEach(key => console.log(key));

// Using Object.values()
Object.values(settings).forEach(value => console.log(value));

// Using Object.entries()
Object.entries(settings).forEach(([key, value]) => console.log(`${key}: ${value}`));

Object Spread and Rest Operators

  • Spread operator (...) allows for shallow copies and merging of objects.
  • Rest operator collects remaining properties into a new object.

let defaults = { theme: "light", notifications: true };
let userSettings = { notifications: false };

let settings = { ...defaults, ...userSettings };
console.log(settings); // Output: { theme: "light", notifications: false }

let { theme, ...rest } = settings;
console.log(theme); // Output: light
console.log(rest); // Output: { notifications: false }

Built-In Object Methods

  • Object.assign(): Copies properties from one or more source objects to a target object.
  • Object.freeze(): Freezes an object, preventing any modifications.
  • Object.seal(): Seals an object, allowing changes to existing properties but preventing new properties from being added.
  • Object.keys(), Object.values(), Object.entries(): Useful for iterating over object properties.

let target = { a: 1 };
let source = { b: 2 };

Object.assign(target, source);
console.log(target); // Output: { a: 1, b: 2 }

let frozenObject = Object.freeze({ x: 10 });
// frozenObject.x = 20; // Error: Cannot assign to read-only property 'x'

let sealedObject = Object.seal({ y: 20 });
sealedObject.y = 30; // Works
sealedObject.z = 40; // Error: Cannot add property 'z'

Common Mistakes with Objects

  • Accidentally mutating objects when intended to copy.
  • Confusing this context in object methods, especially in nested functions or callbacks.
  • Using for...in with arrays, leading to unexpected behavior.

let user = { name: "Alice" };
let copy = user; // Not a copy, but a reference
copy.name = "Bob";
console.log(user.name); // Output: Bob (both point to the same object)

let correctCopy = { ...user }; // Creates a shallow copy
correctCopy.name = "Charlie";
console.log(user.name); // Output: Bob

Best Practices

  • Use object literals for easy and readable object creation.
  • Prefer const for objects that should not be reassigned.
  • Use Object.freeze() or Object.seal() for immutable or semi-immutable objects.
  • Use object destructuring for clear and concise variable assignments.
  • Always handle this carefully within object methods, using arrow functions or .bind() as needed.

Summary

  • Objects are key-value collections that can store data and functions.
  • They are used to represent entities, data structures, and more in JavaScript.
  • Understanding objects and their methods is essential for effective JavaScript programming.
  • Best practices help ensure your code is maintainable, readable, and efficient when working with objects.
